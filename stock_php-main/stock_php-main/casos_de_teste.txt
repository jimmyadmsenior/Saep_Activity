------------------------------------------------------------------------------
CT001 - TESTE DE AUTENTICACAO COM CREDENCIAIS VALIDAS
------------------------------------------------------------------------------
Requisito Funcional: RF001, RF002, RF004
Pre-condicoes: Usuario cadastrado no banco de dados
Passos:
  1. Acessar a pagina de login (login.php)
  2. Inserir usuario: "ana.silva"
  3. Inserir senha: "123456"
  4. Clicar no botao "Entrar"
Resultado Esperado: Usuario e autenticado e redirecionado para interface principal
Pos-condicoes: Sessao do usuario e criada, nome do usuario e exibido no header

------------------------------------------------------------------------------
CT002 - TESTE DE AUTENTICACAO COM CREDENCIAIS INVALIDAS
------------------------------------------------------------------------------

CT005 - CADASTRO DE NOVO LIVRO COM DADOS VALIDOS
------------------------------------------------------------------------------

  1. Clicar em "Novo Livro"
  2. Preencher titulo: "Teste de Software"
  3. Preencher autor: "Autor Teste"

Pos-condicoes: Livro aparece na listagem de livros

CT012 - REGISTRO DE ENTRADA DE ESTOQUE

==============================================================================
8.2 - FERRAMENTAS E AMBIENTES DE TESTE
==============================================================================

AMBIENTE DE DESENVOLVIMENTO/TESTE:
----------------------------------
- Sistema Operacional: macOS Darwin 24.6.0
- Servidor Web: Apache 2.4 ou PHP Built-in Server
- PHP: Versao 7.4 ou superior
- MySQL: Versao 5.7 ou superior
- Navegador: Google Chrome, Firefox ou Safari (versoes recentes)

FERRAMENTAS UTILIZADAS:
-----------------------
1. Servidor de Desenvolvimento:
   - PHP Built-in Server (php -S localhost:8000)
   - XAMPP / MAMP / WAMP (conforme sistema operacional)

2. Gerenciador de Banco de Dados:
   - phpMyAdmin (interface web)
   - MySQL Workbench (aplicacao desktop)
   - HeidiSQL (Windows)

3. Editor de Codigo:
   - Visual Studio Code
   - PHPStorm
   - Sublime Text

4. Teste de Interface:
   - Navegadores com DevTools para inspecao
   - Teste manual de formularios e validacoes
   - Teste de responsividade (diferentes resolucoes)

5. Teste de Banco de Dados:
   - Queries diretas via phpMyAdmin/Workbench
   - Verificacao de integridade referencial
   - Teste de constraints e foreign keys

METODOLOGIA DE TESTE:
---------------------
- Teste Manual de Interface (UI Testing)
- Teste Funcional (Functional Testing)
- Teste de Integracao (Integration Testing)
- Teste de Validacao de Dados (Data Validation Testing)
- Teste de Sessao e Autenticacao (Authentication Testing)
- Teste de Banco de Dados (Database Testing)

CRITERIOS DE ACEITACAO:
-----------------------
- Todos os requisitos funcionais devem ser atendidos
- Sistema deve ser intuitivo e facil de usar
- Validacoes de dados devem funcionar corretamente
- Alertas de estoque minimo devem ser acionados automaticamente
- Historico de movimentacoes deve ser completo e rastreavel
- Nao deve haver erros de SQL ou PHP durante a execucao
- Interface deve ser responsiva e funcionar em diferentes navegadores
